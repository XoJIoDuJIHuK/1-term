Sass - Syntactically Awesome Style Sheets(синтаксически привлекательные таблицы стилей)
Препроцессор - программа, имеющая собственный синтаксис, который затем компилируется в стандартный CSS-код
Sass - препроцессор CSS, который представляет собой метаязык на основе CSS,предназначенный для расширения его возможностей и упрощения каскадных таблиц стилей

Преимущества:
1) Совместимость с различными версиями CSS
2) Функциональность (расширение возможностей CSS, добавление функций, которых в CSS нет)
3) Позволяет использовать конструкции из программирования для разработки кода CSS
4) Совместимость с CSS-фреймворками (зарание готовыми решениями) 
5) Организация CSS-кода в файлы меньшего размера
6) Устранение трудностей, свзяанных с обновлением свойств
7) Уменьшение размера кода

Основы Sass
Правила Sass/Css можно разделить на несколько файлов, а затем объединиить их в один CSS-файл. Чтобы сообщить процессору Sass, что фрагменты не должны преобразовываться в отдельные файлы,
их имена должны начинаться с _
@import "_first.scss"
@import "_second.scss"
@import "_third.scss"
ИЛИ
@import "first"
@import "second"
@import "third"

Переменные Sass - атрибуты, которым может присваиваться значение, которое может изменяться
$font_color: #083B91;
h1 {
	color: $font_color;
}

Примеси

Мини-программы, которые представляют собой ссылки на группу объявления свойств, применяемы далее многократно
Интерполяция - получение нового значения, используя другие

Функции - отношения между элементами, при которых изменение в одном влечёт изменение в другом
