XML - расширяемый язык разметки, созданный для передачи, хранения, транспортировки, обмена данными. Является не исполняемым кодом, а языком описания данных.
Отличия от ХТМЛ: в XML позволено создавать собственные теги. Используется для создания баз данных, приложений (для доступа к данным).
Есть несколько расширений.
Версия 1.0 - 1998, 1.1 - 2006
Предназначен для: 1) отделения данных от ХТМЛ, 2) упрощения распределения данных, 3) для обмена данными между программами, написанными на разных языках, 4) для выгрузки и перезагрузки баз
    данных.
ХМЛ-документ состоит из атрибутов, элементов, сущностей и инструкций обработки. Первые строки до корневого элемента называются прологом документа и сообщают о том, что это ХМЛ-документ и
    какой он версии.
Первая строка содержит с объявления ХМЛ. Существуют три атрибута, значения которых могут быть установлены в этом объявлении: version, encoding - кодировка, standalone - присутствуют ли
    другие элементы, которые необходимо загрузить.
Вторая строка содержит объявление тпиа документа ХМЛ, например, <!DOCTYPE корневой элемент [внутреннее подмножество]>. Может включать в себя файлы, находящиеся в любом месте интернета
Правила ХМЛ-элементов:
	1) имена могут содержать буквы, числа и другие симаволы
	2) имена не могут начинаться с цифры, символа пунктуации или XmL в любом регистре любой буквы
	3) имена не могут содержать пробелы
Атрибуты ХМЛ могут иметь уникальные имена, но некоторые зарезервированы, например, xml:lang классифицирует элемент по языку, на котором написано его содержимое, xml:space определяет,
    должны ли сохраняться в содержимом пробелы, xml:link сообщает, что элемент является элементом ссылки, xml:attribute позволяет переназначить эти "специальные" атрибуты
Правила ХМЛ
1) все теги должны закрываться
2) имена элементов регистрозависимы
3) ХМЛ должны соблюдать корректную вложенность
4) ХМЛ-документ должен содержать корневой элемент
5) значнеия атрибутов должны заключаться в кавычки
6) комментарии можно помещать в любое место документа, но не перед объявлением ХМЛ и не внутри тегов; анализатор ХМЛ их игнорирует
Стандарты ХМЛ - набор расширений, которые придают файлам ХМЛ дополнительные возможности:
	1) XSL - таблицы, предназначен для визуализации элементов ХМЛ
	2) XML Schema
	3) RelaxNG - простой язык схем, разработнный в качестве альтернативы ХМЛ Схеме. В нём документ ХМЛ проверяется по определениям типов данных, импортированных из других источников
	4) Schematron - его реализация написана на ХЛТ. В нём присутствует всего 6 элементов, в нём присутствует Хпатх
	5) SAX - определяет события и интерфейсы для взаимодействия парсером SAX
	6) XSLT
	7) Xpath
	8) SVG
	9) Xlinks определяет ссылки на другие веб-ресурсы
	10) Xpointer - (ссылки?) внутри самого документа
	11) AJAX - совокупность ДжаваСкрипт и ХМЛ
Схема DTD (document type definition) - язык описания структуры ХМЛ-документа, который используется для проверки грамматики ХМЛ-документа и его соответствия содержимому
	Инструкция ADDLIST используется для перечисления и объявления всех атрибутов, которые могут принадлежать элементу. Сначала указывается имя элемента(ов), к которому относится список
    атрибутов. Затем для всех атрибутов по очереди указывается имя, обязательность и символьные данные, допустимые в качестве значения
Типы атрибута DTD CDATA - атрибут содержит только символьные данные, ID, IDREF, ENTITY - значение атрибута содержит несколько имён сущностей, ENTITIES, NMTOKEN
Инструкция ELEMENT применяет для объявления каждого элемента, который используется внутри типа документа, определённого в ДТД. Сначала инструкция определяет имя элемента,
Инструкция NOTATION 
Инструкция ENTITY используется для определения сущностей в ДТД с целью их использования как в связанном с ДТД ХМЛ-документе, так и собственно в ДТД.
ХМЛСхема - современная альтернатива ДТД, основанная на ХМЛ. Определяет элементы и атрибуты, которые могут появляться в документе, какие элементы могут являться дочерними, последователь-
    ность, в которой появляютя дочерние элементы и их число, определяет, пустой ли элемент или он может включать в себя текст, определяет типы данных, элементов и атрибутов.
